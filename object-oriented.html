Lessons learnt and key outcomes

Throughout this module, I developed a comprehensive understanding of object-oriented programming and system design, beginning with the fundamentals of Python and progressing through advanced topics such as abstraction, inheritance, and polymorphism. I learned to critically assess information systems, drawing lessons from case studies involving system failures, and applied this knowledge to create effective UML diagrams - including use case, activity, state machine, and class diagrams - to support system planning. These skills enabled me to design and implement structured, scalable applications in Python. I also gained practical experience with data structures, search algorithms, testing tools, and design patterns, while appreciating the value of sustainable, maintainable code. Overall, the module equipped me with the technical and analytical tools needed to build robust software systems that support business decision-making and operational efficiency.

Unit 1: An Introduction to Python Programming and the OO Programming Paradigm

In this unit, I had the opportunity to explore the foundational principles of Python programming alongside the key concepts underpinning object-oriented programming (OOP). I examined the evolution of programming paradigms and learned how Python incorporates OOP features such as encapsulation, inheritance, abstraction, and polymorphism. Through practical activities and guided readings, I gained hands-on experience with creating Python classes and working with various data types and access modifiers. I also participated in a collaborative discussion, where I reviewed and provided constructive feedback on my peers' contributions, deepening my understanding of OOP concepts through reflection and dialogue.

Unit 2: Object Oriented Analysis - Initial Steps towards Programming in Python

In this unit, I took my first steps into analysing program domains using object-oriented techniques before beginning Python development. As part of the ongoing collaborative discussion, I engaged with peers' reflections from Unit 1, offering constructive feedback while deepening my grasp of modelling concepts. Unit 11: Pointers, References, Memory, and Design Patterns

In this unit, I explored deeper concepts related to memory management and code architecture within Python. I gained an understanding of how Python simplifies development by abstracting away the need for pointers—unlike languages such as C, which require explicit memory management. This comparison helped me appreciate Python’s role in supporting efficient and sustainable programming. I also investigated design patterns—creational, structural, and behavioural—as a way of organising object-oriented code in reusable and maintainable ways. These patterns provide developers with blueprints to solve common design problems effectively. Additionally, I reflected on the environmental impact of software and how sustainable coding practices, like modular design and efficient resource use, contribute to reducing a system’s carbon footprint. These insights were invaluable in completing the summative system implementation, where I applied design patterns to structure my final Python program.
