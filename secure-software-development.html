Lessons learnt and key outcomes

Through this module, I have developed a security-first mindset, transitioning from basic Python programming to architecting resilient, distributed systems. Key outcomes include mastering UML modelling for risk assessment, implementing cryptographic libraries, and automating quality assurance with linters. I gained a critical understanding of the trade-offs between monolithic and microservice architectures while aligning development with ISO/IEC and OWASP standards. Ultimately, I am now equipped to design, test, and defend software within a modern, risk-aware organizational culture.

Unit 1: Introduction to Secure Software Development

In this unit, I established a foundation in Secure Software Development by critically evaluating the transition from traditional Waterfall methodologies to modern Agile frameworks. I explored how security integration differs across these paradigms and investigated industry-standard frameworks, including OWASP guidelines, to identify and mitigate common code vulnerabilities. A key practical component involved using Unified Modelling Language (UML) to decompose complex activities into structured flowcharts, facilitating secure design from the outset. Furthermore, I examined the human element of cybersecurity, gaining an appreciation for the necessity of fostering a risk-aware culture within organizations to complement technical security measures.

Unit 2: UML Modelling to Support Secure System Planning

In this unit, I advanced my practical skills by using UML flowcharts to model software processes and identify potential security vulnerabilities. I focused on decomposing complex logic to communicate architectural risks effectively within the SDLC. Additionally, I researched ISO/IEC standards to align my technical vocabulary with global security management benchmarks. Through a Scrum security review, I explored integrating defensive practices into Agile sprints. I also engaged in a collaborative peer-review process, evaluating flowchart designs and providing constructive feedback, which strengthened my ability to critically analyze and improve system resilience during the design phase.

Unit 3: Programming Languages: History, Concepts & Design

In this unit, I explored the evolution of programming languages, tracing their development from early machine code to modern high-level paradigms. I examined the foundational work of Alan Turing and Alonzo Church, analyzing how their theories influenced language design and the shift toward object-oriented programming (OOP). Using Python, I investigated core concepts such as inheritance, polymorphism, and abstraction, while also identifying common security vulnerabilities and best-practice mitigations. Additionally, I studied design patterns as essential tools for creating scalable, secure software. Finally, I synthesized my learning through a collaborative discussion summary, reflecting on peer feedback to deepen my technical understanding.

Unit 4: Exploring Programming Language Concepts

In this unit, I explored the technical application and security implications of regular expressions (regex) and recursion. I investigated how to implement these concepts within software solutions while critically analyzing potential risks, such as Regular Expression Denial of Service (ReDoS) and stack overflow vulnerabilities. Through hands-on programming exercises, I practiced balancing algorithmic efficiency with system stability. By evaluating these tools against specific security requirements and risk appetites, I gained the expertise to recommend when to utilize these powerful techniques and when to opt for safer alternatives to ensure robust, secure application performance.

Unit 5: An Introduction to Testing

In this unit, I explored the essential role of software testing in ensuring both general quality and robust security. I examined industry-standard frameworks, including OWASP and ISO/IEC/IEEE 29119, to understand how structured testing plans can identify common attack surfaces and minimize system vulnerabilities. I investigated a diverse range of techniques, from functional and usability testing to more specialized penetration and vulnerability assessments. Additionally, I gained practical experience with Python-based automation tools, such as Bandit for security linting and metrics like Cyclomatic Complexity to evaluate code maintainability. This comprehensive approach taught me how to design effective test suites that proactively defend against potential security breaches.

Unit 6: Using Linters to Support Python Testing

In this unit, I explored the practical application of static analysis tools and linters to ensure the development of high-quality, secure Python code. I gained hands-on experience using tools such as PyLint, Flake8, and Bandit to identify logical errors, stylistic inconsistencies, and security vulnerabilities. By comparing and contrasting the outputs of various linters, I learned to appreciate their unique contributions to the development lifecycle and how they enforce adherence to PEP 8 standards. This unit culminated in the creation of a Development Team Project Design Document, where I applied these quality assurance techniques to ensure our code was error-free, consistent, and resilient against common security exploits.

Unit 7: Introduction to Operating Systems

In this unit, I explored the critical relationship between operating systems, programming languages, and system security. I examined core OS functions, specifically distinguishing between processes, threads, and schedulers, and analyzed how application interactions with the kernel can introduce vulnerabilities. I investigated the security implications of using static versus dynamic and shared libraries, learning how to mitigate risks associated with third-party software dependencies. Additionally, I compared different virtualization approaches and their impact on system isolation. Through practical activities, such as exploring a Python-based shell and developing an API for distributed environments, I gained a deeper understanding of how to implement secure protocols and harden an operating system against unauthorized access.

Unit 8: Cryptography and Its Use in Operating Systems

In this unit, I explored the fundamental principles of cryptography and its practical application within modern operating systems. Through a detailed case study of TrueCrypt, I analyzed how cryptographic extensions provide essential data security while also evaluating the potential vulnerabilities and maintenance challenges associated with such tools. I investigated various Python cryptographic libraries, comparing their strengths and weaknesses for different security requirements. This unit included a hands-on programming exercise where I utilized these libraries to develop an application capable of securely encoding data. By participating in collaborative discussions and technical tasks, I deepened my understanding of how to implement robust encryption standards to protect information at the system level.

Unit 10: From Distributed Computing to Microarchitectures

In this unit, I explored the architectural evolution of software systems, tracing the transition from monolithic deployments to microservices, virtualization, and containers. I analyzed how shifting toward distributed modes of operation expands the available attack surface, necessitating advanced security techniques like robust encryption key distribution. I specifically examined the vulnerabilities inherent in Service-Oriented Architecture (SOA) and the unique security threats facing virtualized environments. Furthermore, I investigated the role of ontologies, such as OWL-SOA, in supporting service discovery. This unit concluded with a practical API demonstration, where I integrated these architectural insights to enhance the scalability and security of my technical project.

Unit 11: Future Trends in Secure Software Development

In this unit, I explored the emerging landscape of secure software development, focusing on the shift toward Fog Computing, the Internet of Things (IoT), and Cyber-Physical Systems (CPS). I investigated the unique security challenges posed by these decentralized architectures, where the integration of physical and digital components expands the potential for critical system compromises. A key highlight of this unit was the debate between monolithic and microkernel designs, where I analyzed which approach better supports long-term security and modularity. This forward-looking perspective allowed me to identify potential research fields and recommend emerging technologies to mitigate the risks of future computing environments. I concluded the unit by completing my individual coding project, applying comprehensive testing to validate the security of my own software output.

Unit 12: The Great Tanenbaum-Torvalds Debate Revisited

In this unit, I explored the classic architectural conflict between monolithic kernels and microkernels by revisiting the historic Tanenbaum-Torvalds debate within a modern context. I analyzed how early design trade-offs—such as context switch times and memory splits—have evolved in an era of cloud computing and persistent cyber threats. By evaluating the relationship between microservices and microkernels, I developed a strategic recommendation on which methodology best supports system resilience today. Additionally, I investigated faceted data approaches as a method for enhancing data security. This final unit served as a capstone for the module, allowing me to synthesize my learning into a comprehensive e-portfolio that reflects my growth in designing and advocating for secure software architectures.
